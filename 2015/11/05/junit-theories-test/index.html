JUnit Theories Test 1 : Introduction

05 November 2015 on java, junit, test

Les test de théories. Mais qu'est-ce donc ? A quoi servent-ils ?
Un cas simpliste serait le suivant : Quelque soit 2 entiers positifs a et b, a + b > = a;
Que peut-on écrire comme test pour vérifier cela ?

test 1 : 

@Test
public void should_pass_for_a_and_b_greater_than_0() {  
    int a = 2;
    int b = 4;
    assertThat(a+b > a).isTrue();

}

Remarquer l'utilisation du framework assertJ !!
Dans ce test on ne vérifie que l'assertion pour un couple de valeur précis. On doit surement pouvoir mieux faire (avec une boucle par exemple ?)

test 2 : 

@Test
public void should_pass_for_a_and_b_greater_than_0() {  
    List<Integer> values = Arrays.asList(1, 23, 546, 2034, 20566);
    for(Integer a : values) {
        for(Integer b : values) {
            assertThat(a+b > a).isTrue();
        }
    }
}

Déjà un peux mieux, on vérifie maintenant la théorie pour plusieurs couple de valeurs.
Néanmoins on peut utiliser réecrire notre test2 avec les théories de Junit de la facon suivante :

test JUnit Theories : 

@RunWith(Theories.class)
public class OurMethodTheoriesTest {

    @DataPoints
    public static int[] positiveIntegers() {
        return new Int[]{1, 23, 546, 2034, 20566};
    }

    @Theory
    public void should_pass_for_a_and_b_greater_than_0(Integer a, Integer b) {
        assertThat(a+b > a).isTrue();
    }
}

Commentons un peu ce test :

  • @RunWith(Theories.class) : Cette annotation dis a JUnit d'utiliser le runner pour les test de theories, sans celle-ci on peut utiliser les test de theories.
  • @Datapoints : Cette annotation permet de définir la méthode qui alimentera nos test en données. Dans cette exemple, cette methode renvoie des int.
  • @Theory : Cette annotation remplace la classique @Test. La theorie prends des parametres en entree et est executee avec TOUTES les combinaisons possibles de parametres. Ainsi dans notre exemple le test sera lance 5^2 fois !! (avec [1,1] puis [1,23] puis [1,546] ...)
